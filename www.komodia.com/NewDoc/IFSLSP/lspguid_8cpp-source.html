00010 // Description:
00011 //
00012 //      This module defines the GUID under which the LSP's hidden dummy
00013 //      entry is to be installed under. This GUID must be hard coded in
00014 //      this file so that the DLL instance can find itself in the Winsock 
00015 //      catalog. Once it finds the dummy hidden entry, it can then find
00016 //      the layered protocol entries belonging to the LSP (since these
00017 //      entries will reference the catalog ID of the hidden dummy entry).
00018 //      
00019 //      It is possible to create multiple LSPs by simply copying this project
00020 //      to another directory, changing the GUID in this file, and changing
00021 //      the DLL name. The LSP installer (instlsp.exe) can then install each
00022 //      LSP with the -d flag which is the full path to each LSP to install.
00023 //
00024 //      A new export has been added: GetLspGuid. This allows the LSP installer
00025 //      to install an arbitrary LSP as long as it exports this function as the
00026 //      installer needs to find the GUID to install the hidden dummy entry
00027 //      with. The GUIDs for the layered protocol entries are generated on the
00028 //      fly.
00029 //
