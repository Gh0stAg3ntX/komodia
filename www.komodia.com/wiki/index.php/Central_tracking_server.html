Central tracking server

From Komodia
Jump to: navigation, search

Komodia's Central tracking server is an optional module of Komodia's Redirector and it is used to see remote clients' settings, view who's online and update the clients' settings remotely.

How does it work

  • The server listen on an predetermined IP and port 443 waiting for clients to connect to it, the session is protected by SSL with certificates validation. Both the server and client validate that the remote side is using their certificate, if the certificate is wrong, the session will not be established.
  • As soon as a client is active it will try to connect to the server (the IP/domain is compiled into the code), if the server is offline a connection attempt will be made every five minutes.
  • When a connection has been established the client will send its configuration to the server.
  • When a client had its data changed it will update the server with the new configuration.
  • The operator on the server can see the list of all online clients.
  • The operator can then select a client and inspect its configuration.
  • When updating a remote configuration the operator will receive confirmation if the settings was saved successfully.

Install

Please note you can't install the server and client on the same machine.

Server

Setup the ini file

  • The ini file name is: PCProxy.ini and it must be in the directory of the Redirector, if there's no INI file the Redirector will bind to all interfaces. Listen port is 443 (always).
  • You need to set the IP that the Redirector will bind to, this is a sample ini file: 
[bind]

ip=192.168.0.1
  • Install the Redirector: 
PCProxy /Auto /Service
sc start pcproxy

Client

There's no special install instructions for the client, you need to follow the normal Komodia's Redirector installation guide.

Usage guide

Viewing online client

  • To view the list of current online clients press the "Online clients" button on the main console's page.
  • It is possible to configure the list to show one custom parameter from the client machine (for example an email or serial number), to do so add a custom variable called: "flagtouse" with the name of the custom variable you want to view (make sure to save the settings)

Working with a remote client

  • In the online clients screen select a client and then choose: "Select client", the name of the client will appear in the main console.
  • In the main console, every operation will now be made on the local copy in the server representing the data, you can view and change the data.
  • When you want to "push" the settings to the remote client press the button: "Save remote" after which you will receive a confirmation of a success or a failure.
